# Runbook: служба поддержки Делифон

## Каналы обращений

| Канал | SLA первого ответа |
|-------|-------------------|
| Чат в приложении | 15 мин (9:00–22:00) |
| Telegram-бот | 30 мин |
| Email | 4 часа |

## Типовые сценарии

### 1. Клиент не может разблокировать устройство
- Убедись, что клиент вводит правильный PIN (последние 4 цифры телефона).
- Если не помогает — инициируй удалённый сброс через MDM.
- Эскалация: технический отдел.

### 2. Клиент потерял устройство
- Создай инцидент типа `lost` через админку.
- Уведоми клиента о возможном удержании залога.
- Активируй геолокацию устройства (если доступна).

### 3. Повреждение при возврате
- Оператор фиксирует фото повреждений.
- Создаётся инцидент типа `damage`.
- Автоматический расчёт удержания по таблице damage_pricing.
- Клиент получает уведомление с суммой.

### 4. Спор по списанию
- Проверь историю платежей в разделе «Финансы» клиента.
- При ошибочном списании — инициируй возврат через ЮKassa.
- Зафиксируй решение в чате.

## Эскалация

- Уровень 1: оператор поддержки (типовые вопросы).
- Уровень 2: старший оператор (споры, возвраты > 1000 ₽).
- Уровень 3: менеджер (юридические вопросы, блокировки).

## Метрики

- CSAT > 4.5 / 5
- Среднее время решения < 2 часов
- Процент эскалаций < 10%
